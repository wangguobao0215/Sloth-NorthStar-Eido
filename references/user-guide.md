# Sloth-NorthStar-Eido 用户指南

> 版本：0.2.0 | 适用于 QoderWork 平台

---

## 1. 技能定位

Sloth-NorthStar-Eido 是一套面向**滑铁卢大学 CS Co-op 学生**的全人发展 Skill 家族。它不是单一技能，而是由 5 个独立子技能组成的协作系统，覆盖四个维度：

- **AI 学习导航** -- 系统化 AI 学习路径规划与进度追踪
- **身体健康守护** -- 睡眠、锻炼、精力与维生素 D 追踪
- **心理韧性支持** -- 情绪记录、正常化支持与专业资源导引
- **职业发展助手** -- 简历优化、面试准备与 Co-op 求职策略

核心差异化能力是 **Co-op 周期感知**：根据学生所处的 Study Term、Co-op Term、求职季或考试周，自动调整所有子技能的行为强度和关注重点。

**目标用户**：滑铁卢大学计算机科学专业的 Co-op 学生，尤其是希望在高强度学业和求职压力下实现 AI 自学的学生。

## 2. 安装与初始化

### 2.1 安装方法

每个子技能可独立安装，按需选择：

```bash
# 推荐首先安装学习导航
cp -r sloth-northstar-learn ~/.qoderwork/skills/

# 按需安装其他子技能
cp -r sloth-northstar-health ~/.qoderwork/skills/
cp -r sloth-northstar-mind ~/.qoderwork/skills/
cp -r sloth-northstar-career ~/.qoderwork/skills/
cp -r sloth-northstar-core ~/.qoderwork/skills/
```

**安装建议**：
- 如果你只关心 AI 学习，安装 `learn` 即可
- 如果你想获得完整体验，全部安装后它们会通过共享数据协同工作
- `core` 负责全局画像管理和学期切换，安装全部子技能时建议一并安装

### 2.2 首次使用

安装后在 QoderWork 中正常对话即可，子技能会在你提到相关话题时自动触发。首次使用时，Skill 只会问 2-3 个简短问题了解你的情况：

- 你的名字、年级、当前学期类型
- 每周可用于 AI 学习的时间
- 偏好的学习模式

没有冗长的设置流程。信息会在后续对话中自然积累。

### 2.3 数据存储

所有子技能通过共享数据目录协同：

```
~/.qoderwork/skills/sloth-northstar-data/
├── user_profile.json        # 共享用户画像
├── learning_progress.json   # 学习数据（learn 专属写入）
├── health_log.json          # 健康数据（health 专属写入）
├── mood_log.json            # 情绪数据（mind 专属写入）
└── career_tracker.json      # 求职数据（career 专属写入）
```

## 3. 核心功能详解

### 3.2 Learn -- AI 学习导航 (v0.2.0)

Learn 是家族中功能最丰富的子技能，包含 18 个工作流，支持三种学习模式。

#### 三种学习模式

| 模式 | 理念 | 适合人群 |
|:---|:---|:---|
| **Bottom-Up（系统路线）** | 先打地基，再做应用 | 喜欢先理论后实践的学习者 |
| **Top-Down（项目驱动）** | 先做项目，遇到什么补什么 | 做中学、实战驱动的学习者 |
| **Hybrid Spiral（混合螺旋）** | 理论与项目交替推进 | 过了入门阶段的大多数学习者 |

#### 课程体系

**Bottom-Up 路径（4阶段，18-30个月）：**

| 阶段 | 重点 | 时长 | 核心内容 |
|:---|:---|:---|:---|
| Phase 1 地基 | Python、ML基础、数学 | 4-6月 | Scikit-learn、线性代数、概率论 |
| Phase 2 核心 | 深度学习 | 6-9月 | CNN、RNN、Transformer、从零构建LLM |
| Phase 3 前沿 | 生成式AI与LLM | 6-9月 | LLM原理、Prompt Engineering、RAG、Agent、MCP |
| Phase 4 工程化 | MLOps | 3-6月 | 模型服务、ML CI/CD、监控 |

**Top-Down 路径（5级项目驱动）：**

| Level | 项目目标 | 预计时长 |
|:---|:---|:---|
| Level 1 | AI 聊天机器人 | 0-3个月 |
| Level 2 | RAG 文档问答系统 | 3-6个月 |
| Level 3 | AI Agent + MCP | 6-9个月 |
| Level 4 | LLM 微调/从零训练 | 9-12个月 |
| Level 5 | 多Agent平台 | 12个月+ |

种子课程库包含 **35门免费课程**，特别关注 AI Agent（7门）、MCP（3门）、LLM从零构建和微调方向。

#### v0.2.0 特色功能

- **姓名个性化**：系统记住名字，建立学长型导师关系
- **自适应难度校准**：根据实际完成时间 vs 预估时间自动调整学习节奏
- **课程收集者防御**：活跃课程限制2门，检测放弃模式时发出警告
- **知识雷达图**：ASCII 可视化12+个AI领域的掌握程度
- **认知负荷管理**：新概念学习45分钟封顶，能量感知推荐
- **论文阅读阶梯**：3级台阶（博客解读 -> 经典论文关键章节 -> 独立读新论文）
- **Schema自动迁移**：跨版本数据格式自动升级（1.0 -> 1.3），无需用户干预

### 3.3 Health -- 身体健康守护 (v0.1.0)

定位为"硬件基础设施守护者"，通过对话式自报告追踪身体状态。

**核心功能：**

| 功能 | 说明 |
|:---|:---|
| 日常健康签到 | 极简签到：睡眠时长、精力评分、是否锻炼 |
| 趋势分析与预警 | 连续3天睡眠不足6小时触发黄色预警，连续7天触发红色预警 |
| 冬季模式 | 11月-3月自动启用，提高维D和光照关注度 |
| 每周健康回顾 | 每7天生成睡眠、锻炼、精力趋势报告 |
| 固定锚点保护 | 当用户想压缩睡眠换学习时间时，提供替代方案 |

**使用示例：**
- "昨晚只睡了5小时" -> 记录并观察趋势
- "这周没怎么锻炼" -> 建议微运动方案
- "最近精力特别差" -> 综合分析睡眠+运动数据找原因

### 3.4 Mind -- 心理韧性支持 (v0.1.0)

严格定位为"指路牌"而非"治疗师"，提供情绪追踪和专业资源导引。

**核心功能：**

| 功能 | 说明 |
|:---|:---|
| 每日情绪签到 | 1-5分情绪打分，附带上下文记录 |
| 支持性对话 | 三步走：承认感受 -> 正常化 -> 引导行动 |
| 危机信号响应 | 检测到危机信号时立即提供专业资源 |
| 意义锚点管理 | 帮助学生找到和维护"为什么选择CS"的内在动力 |
| 情绪趋势回顾 | 每2周提供情绪趋势分析 |

**常驻专业资源：**

- Good2Talk：1-866-925-5454（安省学生专线）
- UW Counselling Services：519-888-4567 x32655
- Crisis Text Line：发送 CONNECT 至 686868

### 3.5 Career -- Co-op 求职助手 (v0.1.0)

专为滑铁卢 Co-op 求职体系设计，理解加拿大雇主偏好和 WaterlooWorks 节奏。

**核心功能：**

| 功能 | 说明 |
|:---|:---|
| 简历优化 | 使用 CAR 法则（Challenge-Action-Result）将"任务描述式"转化为"价值导向式" |
| 面试准备 | 公司面试风格解码、STAR 行为面准备、技术面刷题规划 |
| 职业叙事构建 | 串联零散 Co-op 经历，生成连贯的职业成长故事线 |
| 求职季模式 | 投递策略、每周 check-in、被拒后的具体改进建议 |
| WaterlooWorks 策略 | 分层投递建议、Ranking 策略、时间节点提醒 |

**使用示例：**
- "帮我把这段经历写到简历上" -> CAR 法则转化
- "我要面试Google" -> 该公司面试风格解码+针对性准备清单
- "被拒了好几个" -> 正常化支持+具体改进方向

### 3.6 Core -- 画像与协调中心 (v0.1.0)

家族的"中控台"，负责用户画像管理、学期切换和全局状态总览。

**核心功能：**

| 功能 | 说明 |
|:---|:---|
| 首次画像建立 | 渐进式收集用户信息，分2-3次对话完成 |
| 学期类型切换 | 触发所有子技能行为调整的核心操作 |
| 固定锚点管理 | 统一管理睡眠时间、锻炼频率等固定设置 |
| 全局状态总览 | 一屏展示学习、健康、情绪、求职四个维度的状态 |
| Skill安装引导 | 帮助用户了解家族成员并按需安装 |

## 4. 工作流与使用流程

### 4.1 典型使用场景

**场景一：新学期开始**
1. 告诉 Core "新学期开始了"，完成学期切换
2. Learn 自动调整学习强度和课程安排
3. Health 根据新作息调整锻炼建议

**场景二：日常学习**
1. 打开 QoderWork，说"今天学什么"
2. Learn 根据当前进度、能量状态、Co-op阶段推荐今日任务
3. 学习完成后打卡，更新 Streak

**场景三：求职季**
1. Core 切换到求职季模式
2. Learn 暂停常规学习，切换面试准备
3. Career 全面激活：简历优化、面试准备、投递策略
4. Mind 高度关注被拒压力

**场景四：考试周**
1. Learn 完全暂停
2. Health 最低限度保护睡眠和锻炼
3. Mind 压力感知模式

### 4.2 Co-op 周期行为矩阵

| 学期类型 | Learn | Health | Mind | Career |
|:---|:---|:---|:---|:---|
| **Study Term** | 全强度（6-8小时/周） | 常规监测 | 正常运行 | 休眠 |
| **Co-op Term** | 降低强度（2-4小时/周） | 适配工作时间表 | 关注环境适应压力 | 经历积累 |
| **求职季** | 暂停，切换面试准备 | 关注过劳 | 高度关注被拒压力 | 全面激活 |
| **考试周** | 完全暂停 | 最低限度保护 | 压力感知 | 休眠 |

### 4.3 跨子技能协同

子技能之间通过共享数据目录实现协同：

- **Health -> Learn**：当健康数据显示连续睡眠不足时，Learn 自动降低学习强度
- **Mind -> Learn**：当情绪持续低落时，Learn 减少学习提醒频率
- **Learn -> Career**：学习成果自动转化为求职叙事素材
- **Core -> 全部**：学期切换统一触发所有子技能行为调整

## 5. 最佳实践

### 5.1 推荐安装顺序

1. **sloth-northstar-learn** -- 功能最丰富，独立运行价值最高
2. **sloth-northstar-core** -- 如果安装多个子技能，建议一并安装用于协调
3. **sloth-northstar-health** / **sloth-northstar-mind** -- 按需安装
4. **sloth-northstar-career** -- 求职季前安装

### 5.2 日常使用建议

- **每天**：花1分钟和 Learn 对话，获取今日学习推荐，完成后打卡
- **每周**：配合 Learn 的每周回顾，审视本周进度和下周计划
- **学期切换时**：通过 Core 切换学期类型，让所有子技能自动适配
- **保持诚实**：Health 和 Mind 的价值在于真实记录，不需要每天都是满分

### 5.3 避免常见误区

- **不要同时开始太多课程**：Learn 限制活跃课程不超过2门，完成1门的价值远大于开始5门
- **不要压缩睡眠换学习时间**：Health 会帮你找到更好的替代方案
- **不要忽略5分钟模式**：状态不好时，5分钟的学习也算数，保持连续性比单次时长更重要
- **不要期待心理治疗**：Mind 是指路牌，不是治疗师。需要专业帮助时请联系专业资源

## 6. 常见问题（FAQ）

### Q1：我可以只安装其中一个子技能吗？

可以。每个子技能都可独立运行。推荐从 `sloth-northstar-learn` 开始，它的功能最完整。当多个子技能同时安装时，它们通过共享数据提供更丰富的体验（例如 Health 数据影响 Learn 的学习强度建议），但这不是必须的。

### Q2：课程信息是否会过时？

会。AI 领域的课程半衰期约6个月。Learn 子技能内置了课程健康检查机制（每季度1次），会检查链接可达性和内容时效性。但在注册任何课程前，请务必点击链接确认最新状态。

### Q3：我不是滑铁卢大学的学生，能用吗？

技能的核心价值（AI 学习路径规划、健康追踪、情绪支持）对所有学生适用。但 Co-op 周期感知、WaterlooWorks 策略、滑铁卢校内资源等功能是专为滑铁卢学生设计的。其他学校的学生可能需要忽略这些特定内容。

### Q4：数据存在哪里？如何迁移？

所有数据存储在 `~/.qoderwork/skills/sloth-northstar-data/` 目录下，以 JSON 文件形式保存。迁移时只需复制整个目录即可。数据文件支持自动 Schema 迁移（从 v1.0.0 到 v1.3.0），跨版本升级时无需手动处理。

### Q5：健康建议和心理支持靠谱吗？

Health 子技能提供的是通识性健康建议，不是医疗建议。如有持续身体不适，请咨询专业医疗人员。Mind 子技能严格定位为"指路牌"，提供情绪记录和专业资源导引，不进行心理诊断或治疗。检测到危机信号时，第一响应是提供专业心理健康热线和服务。
