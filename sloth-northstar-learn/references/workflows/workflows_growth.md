# 成长加速流程（流程十一~十五）

> 本文件包含学习成果转化与深度巩固流程：求职叙事、学期交接、费曼教学、工作经历学习化、能量感知。
> 这些流程在特定里程碑或条件下触发。

### 流程十一：求职叙事自动生成

**核心理念：学习不是孤立的事，它最终要转化为你在面试中讲的故事。每个阶段的学习成果都应该自动沉淀为求职素材。**

**触发条件：**
- 用户完成一个课程或项目阶段
- 用户主动说"帮我整理求职素材"或"我要准备面试了"
- 每月自动检查一次，如有新的可叙述成果则提醒

**自动生成逻辑：**
1. 从 `learning_progress.json` 中提取已完成课程、已完成项目迭代、知识图谱高掌握度节点
2. 将学习成果转化为 **STAR 格式的叙事片段**：
   - **Situation**：在滑铁卢CS学习期间，为了系统掌握AI能力…
   - **Task**：自主设定了24个月AI学习路线，目标是…
   - **Action**：完成了 [具体课程/项目]，掌握了 [具体技能]…
   - **Result**：能够独立 [具体能力]，构建了 [具体成果]…
3. 按技能方向聚类（LLM应用、Agent开发、MLOps等），生成**技能叙事卡片**

**叙事卡片格式：**
```
📋 技能叙事：RAG应用开发
━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 核心能力：能独立构建基于RAG的文档问答系统
📚 学习基础：完成LangChain课程 + Embedding专题 + 向量数据库实践
🔨 项目证明：[项目名]——[一句话描述]
💬 面试话术：
"我花了3个月系统学习RAG技术栈，从Embedding原理到向量检索，
最终独立完成了一个[项目描述]，支持[具体功能]。"
```

**数据存储：** 存储在 learning_progress.json 的 `job_narratives` 数组中：
```json
{
  "job_narratives": [
    {
      "skill_area": "RAG应用开发",
      "generated_at": "2026-04-18",
      "courses_involved": ["course_id_1", "course_id_2"],
      "projects_involved": ["project_id_1"],
      "star_narrative": "...",
      "interview_pitch": "...",
      "knowledge_nodes": ["RAG", "Embedding", "Vector_DB"]
    }
  ]
}
```

**与NorthStar Career协同：** 如检测到 sloth-northstar-career 已安装，将叙事卡片同步至简历模块。

### 流程十二：学期智能交接备忘录

**核心理念：每次学期切换都是一次上下文断裂。交接备忘录确保"未来的你"能无缝衔接"过去的你"。**

**触发条件：**
- 用户触发Co-op周期切换（流程六）时自动生成
- 学期结束前2周主动提醒："[名字]，这个学期快结束了，要不要生成一份学习交接备忘录？"
- 用户主动请求

**备忘录生成逻辑：**
从 learning_progress.json 和 user_profile.json 中提取以下信息：

**备忘录格式：**
```
📑 学期交接备忘录 | [学期类型] → [新学期类型]
生成日期：YYYY-MM-DD
━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 本学期学习总结
- 学习天数：X天 | 总时长：约Y小时
- 最长连续打卡：Z天
- 完成课程：[列表]
- 进行中课程：[列表+进度百分比]
- 知识图谱新点亮：[列表]

🔖 进行中的事项（下学期继续）
- [课程名]：学到第X章，下次从[具体位置]继续
- [项目名]：完成了[迭代N]，下次目标是[迭代N+1的目标]

🎯 下学期建议
- 学期类型变化：[旧] → [新]，建议学习强度从X小时/周调至Y小时/周
- 优先恢复：[最值得继续的1-2项]
- 可暂缓：[优先级较低的项目]

💡 上学期的洞察
- [学习过程中发现的个人偏好或有效方法]
- [遇到的困难和应对策略]
```

**数据存储：** 存储在 learning_progress.json 的 `semester_handovers` 数组中：
```json
{
  "semester_handovers": [
    {
      "from_term": "co_op",
      "to_term": "study",
      "generated_at": "2026-04-18",
      "summary": {
        "study_days": 45,
        "total_hours_estimate": 90,
        "longest_streak": 23,
        "completed_courses": [],
        "in_progress_items": [],
        "new_knowledge_nodes": []
      },
      "continuation_notes": [],
      "next_term_suggestions": []
    }
  ]
}
```

**新学期恢复：** 当新学期首次对话时，自动读取最近一份交接备忘录，生成恢复摘要：
"[名字]，欢迎回来！上个学期你学到了[摘要]。我建议这学期从[具体建议]开始。要看完整的交接备忘录吗？"

### 流程十三：费曼教学机制

**核心理念：你能教会别人的，才是你真正懂的。Generation Effect——主动产出比被动接收的记忆效果强3倍。**

**触发条件：**
- 用户刚完成一个课程模块或知识点
- 知识闪回（流程九）中用户回答正确后，有20%概率触发深度费曼
- 用户主动说"我来解释一下"或"帮我验证我是否理解了"

**费曼流程：**

1. **出题**（以生活化场景包装）：
```
🎓 费曼时刻：[名字]，假设你室友问你：
"听说你在学AI，[具体知识点]到底是什么东西？能给我讲讲吗？"
请用你自己的话解释，就像跟一个完全不懂的朋友说一样。
```

2. **评估**用户的解释：
   - **核心概念正确** → 点赞 + 补充一个精妙的类比 → 掌握度 +0.1
   - **大方向对但有误区** → "大方向对了！不过有一个地方可以更准确：[纠正]" → 掌握度 +0.05
   - **偏离核心** → "你说的[X]部分是对的，但[核心概念]的关键其实在于…" → 掌握度不变，标记复习
   - **用户说"我讲不出来"** → "没关系，说明这个点还需要再消化一下。核心其实就一句话：[简短解释]" → 标记下次优先复习

3. **记录到知识笔记本**：
   - 无论用户回答如何，都保存一份精炼的知识笔记
   - 笔记包含：用户自己的解释（原文）+ 标准简述 + 关键类比

**数据存储：** 存储在 learning_progress.json 的 `feynman_notes` 数组中：
```json
{
  "feynman_notes": [
    {
      "knowledge_node": "RAG",
      "date": "2026-04-18",
      "user_explanation": "就是让AI先去找相关文档再回答...",
      "standard_summary": "RAG通过检索增强...",
      "analogy": "就像考试时可以翻书，比纯靠记忆靠谱...",
      "assessment": "core_correct",
      "mastery_change": 0.1
    }
  ]
}
```

**与知识闪回联动：** 费曼笔记中用户自己写的解释，会在未来的知识闪回中作为"你上次是这么解释的"提示出现，增强记忆锚点。

### 流程十四：Co-op工作经历学习化

**核心理念：Co-op不只是赚钱，工作中用到的每一个技能都是学习。把工作经验显性化，关联到知识图谱。**

**触发条件：**
- 用户当前处于Co-op Term
- 用户提到工作中的技术话题："今天在公司用了Docker"、"我写了个Python脚本自动化了XX"
- 每周回顾时（流程四）主动询问："这周工作中有用到什么AI相关的技能吗？"

**记录流程：**

1. **捕捉工作中的学习时刻**：
```
🏢 工作经验记录：
[名字]提到在工作中 [用到了XX/遇到了XX问题/学会了XX]

📌 关联知识节点：[自动识别相关的knowledge_map节点]
📈 实践加成：这个节点的掌握度因为实际应用经验 +0.1

💡 这可以怎么说：
"在Co-op期间，在生产环境中实际应用了[技能]来解决[问题]…"
→ 已自动保存到求职叙事素材库
```

2. **自动关联知识图谱**：
   - 识别工作经历中涉及的知识节点
   - 实际工作中使用的技能，掌握度加成比课程学习更高（+0.1 vs +0.05）
   - 因为"做过"比"学过"更有说服力

3. **自动生成求职叙事**：
   - 每次记录工作经历时，同步生成一条STAR格式素材，存入 `job_narratives`
   - Co-op工作经历标记为 `source: "work_experience"`，面试时权重更高

**数据存储：** 存储在 learning_progress.json 的 `work_experience_log` 数组中：
```json
{
  "work_experience_log": [
    {
      "date": "2026-04-18",
      "company": "匿名（用户可选填）",
      "description": "用Python写了个脚本自动化数据清洗流程",
      "skills_used": ["Python", "Data_Preprocessing"],
      "knowledge_nodes_boosted": ["Python", "Data_Preprocessing"],
      "mastery_boost": 0.1,
      "narrative_generated": true
    }
  ]
}
```

**与求职叙事联动（流程十一）：** 工作经历自动汇入求职叙事库，标记为"实战经验"，面试时优先使用。

### 流程十五：学习能量感知

**核心理念：不同时间段适合不同类型的学习。困了硬看数学推导是浪费时间，不如看个轻松的科普视频。**

**能量等级定义：**
- **高能量 🟢**：思维清晰、专注力好 → 适合"深潜"——理论推导、代码实战、新概念学习
- **中能量 🟡**：清醒但不够专注 → 适合"巡航"——看视频教程、做笔记整理、复习
- **低能量 🔴**：疲惫、难以集中 → 适合"漂浮"——听播客、看科普短视频、浏览技术博客

**能量推断逻辑（不问，推断）：**
1. **时间段推断**（主要信号）：
   - 早晨起床后1-3小时 → 通常高能量
   - 下午2-4点 → 通常中/低能量（午后低谷）
   - 晚间锻炼后 → 中/高能量
   - 临睡前1小时 → 低能量
2. **用户状态信号**（辅助信号）：
   - "好累"/"不想动脑" → 低能量
   - "今天状态不错" → 高能量
   - 刚说"锻炼完了" → 中/高能量
3. **历史模式**（长期学习）：
   - 记录用户在不同时段的学习完成率，逐渐学习用户的个人能量曲线
   - 如果某用户总是在晚上10点后高效完成深潜任务 → 不死守"晚上应该低能量"的默认值

**应用方式：**

在推荐学习内容时，自动匹配当前推断的能量等级：

```
🟢 高能量时段推荐：
📖 [课程名] — 第X章：[理论重点] (~1.5h深潜)

🟡 中能量时段推荐：
📺 [课程名] — [视频教程段落] (~45min巡航)

🔴 低能量时段推荐：
🎧 [播客/短视频名] — [主题] (~15min漂浮)
```

**数据存储：** 存储在 user_profile.json 的 `energy_pattern` 字段中：
```json
{
  "energy_pattern": {
    "default_curve": {
      "06-09": "high",
      "09-12": "high",
      "12-14": "medium",
      "14-16": "low",
      "16-18": "medium",
      "18-20": "medium",
      "20-22": "medium",
      "22-24": "low"
    },
    "override_signals": [],
    "completion_history": [
      {
        "date": "2026-04-18",
        "time_slot": "20:30",
        "task_type": "deep_dive",
        "completed": true
      }
    ]
  }
}
```

**不打扰原则：**
- 不直接问"你现在精力怎么样？"——显得像调查问卷
- 通过时间和上下文**静默推断**，直接给出匹配的推荐
- 只有当推断可能不准时才轻轻确认："现在适合看点重的内容，还是想轻松一点？"

**认知负荷管理：**
能量感知不只是"困了看轻松的"，更深层的逻辑是**认知负荷理论（Cognitive Load Theory）**的应用：
- **新概念学习**（intrinsic load高）→ 必须安排在高能量时段，且**限时45分钟**。工作记忆有上限，连续学新东西超过45分钟效率断崖下降
- **实操巩固**（germane load为主）→ 可安排在中等能量时段，不严格限时，因为动手写代码的认知带宽低于纯理论
- **被动浏览**（extraneous load为主）→ 低能量时段，看视频/听播客

**推荐节奏模板：**
```
🟢 高能量深潜（限时45分钟）：
📖 [新概念] — [具体章节] (~45min)
⏸ 然后休息10分钟再决定是否继续

🟡 中能量实操（不限时）：
💻 [练习/代码] — [具体任务] (按需)

🔴 低能量漂浮（15分钟封顶）：
🎧 [播客/视频] — [主题] (~15min)
```
